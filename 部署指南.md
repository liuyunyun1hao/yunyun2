## 🚀 快速部署步骤

### 第一步：准备GitHub仓库

1. 访问 [github.com](https://github.com) 并登录
2. 点击右上角 "+" → "New repository"
3. 填写仓库名称（如 `yunyun-proxy`）
4. 选择 "Public"（公开）
5. 点击 "Create repository"

### 第二步：上传文件

上传以下文件到GitHub仓库：
- `proxy_server_render.py`
- `requirements.txt` 
- `render.yaml`
- `README.md`
- `.env.example`

### 第三步：部署到Render

1. 访问 [render.com](https://render.com)
2. 使用GitHub账号登录
3. 点击 "New +" → "Blueprint"
4. 选择你的GitHub仓库
5. 点击 "Apply"

### 第四步：记录认证信息

部署完成后：
1. 在Render控制台找到你的服务
2. 点击 "Environment" 标签页
3. **记录 AUTH_PASSWORD 的值**（这是你的管理员密码）

### 第五步：导入API密钥

使用以下命令导入硅基流动API密钥：

```bash
curl -X POST 'https://你的服务.onrender.com/api/data' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Basic $(echo -n admin:你的密码 | base64)' \
  -d '{
    "keys": [
      {
        "key": "sk-你的硅基流动密钥",
        "balance": "未知",
        "fail_count": 0
      }
    ],
    "active_key": "sk-你的硅基流动密钥"
  }'
```

## 🔐 安全配置建议

### 方案A: 基础认证（推荐）
- 设置 `REQUIRE_AUTH=true`
- 设置强密码 `AUTH_PASSWORD`
- 在请求头中添加 `Authorization: Basic [base64编码]`

### 方案B: API密钥认证
- 设置 `USE_API_KEY_AUTH=true`
- 设置 `API_KEY=你的自定义密钥`
- 在请求头中添加 `X-API-Key: 你的密钥`

### 方案C: 完全开放（不推荐）
- 设置 `REQUIRE_AUTH=false`
- 任何人都可以访问你的服务

## ⚠️ 重要提醒

### Render免费套餐限制：
- **每月750小时免费时间**（约31天）
- **15分钟不活动后自动休眠**
- **冷启动需要约30秒**
- 适合低流量使用场景
- 如需高可用性，考虑升级到付费计划

### 数据持久性：
- Render免费版重启会丢失本地文件
- 建议定期备份数据：`GET /api/export_backup`
- 或设置 `USE_ENV_STORAGE=true` 将密钥存为环境变量

## 🔧 故障排除

### 服务无法启动：
1. 检查Render构建日志
2. 确认Python依赖安装成功
3. 确认端口配置正确

### 认证失败：
1. 确认设置了正确的环境变量
2. 检查密码是否正确
3. 确认请求头格式正确

### 代理请求失败：
1. 检查API密钥是否有效
2. 确认硅基流动API可达
3. 查看应用日志获取详细信息
